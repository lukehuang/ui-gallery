We namespace the styles so that we can always differentiate between the global Sonatype styles (prefixed with nx-) and specific project styles (for example IQ uses iq-, and FAAS uses io-). No project should ever modify an nx- class. If you need to modify an existing nx- class then do so within your own project's namespace using a --modifier. So for example if you need a custom table cell you should use .nx-cell to get your base styling then add .##-cell--modifier (use your project's namespace in place of ##).
