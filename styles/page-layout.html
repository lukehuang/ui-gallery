We have two basic layouts in IQ Server. A single column and a double column. The double column is used most frequently and consists of a narrow left hand sidebar and a main content area. The single column appears much less frequently, being used on System Preferences pages and in the Success Metrics pages. `.iq-root-container` is present in both single and double pane layouts. It serves to define the maximum page width and keep the panes centered on the page. The main content area of the page is classed with `.iq-body-content`, in a single pane configuration the modifier class `iq-body-content--single-pane` is added. Page specific modifiers should also be created if further custom styles are required. For example: `<main class="iq-body-content iq-body-content--single-pane iq-body-content--page-name">` would result in a single pane layout with further modifications specific to that page. Ideally `.iq-sidebar` and `.iq-body-content` should be applied to their semantically correct HTML5 elements: `<aside>` and `<main>` **Note:** in the examples below the directive `maximize-container-height` is incorrectly displayed as `maximize-container-height=""`. This is a glitch in the code rendering library. The correct usage is simply `maximize-container-height`.
