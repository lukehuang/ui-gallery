Individual _example.scss files will be linked on the example pages. The compiled SCSS for all of the examples can be found here: http://0.0.0.0:4041/assets/nx-styles.css We use Font Awesome ( https://fontawesome.com ) for icons, and in many cases styles are built in to support Font Awesome styles.

The NX Style Gallery

The NX style gallery is where you can find reusable styles (and hopefully soon components) for use in new or existing Sonatype applications. The styles described here are based off the current IQ styling. You can find descriptions of the various styles, including measurements here: https://docs.sonatype.com/display/INSIGHT/UI+Guidelines The purpose of this site is to take those requirements and turn them into HTML/SCSS that can be used in any project.

What's up with the nx- namespace?

We namespace the styles so that we can always differentiate between the global Sonatype styles (prefixed with nx-) and specific project styles (for example IQ uses iq-, and FAAS uses io-). No project should ever modify an nx- class. If you need to modify an existing nx- class then do so within your own project's namespace using a --modifier. So for example if you need a custom table cell you should use .nx-cell to get your base styling then add .##-cell--modifier (use your project's namespace in place of ##).

BEM? What the heck is BEM?

We started using BEM on the IQ project as a standardized naming convention. BEM is pretty simple to understand it stands for Block, Element, Modifier. Here's a simple explainer: https://seesparkbox.com/foundry/bem_by_example and another one: https://css-tricks.com/bem-101/

How to add a new page to the gallery

* create an html page using the following structure (see template.html) ```html <item-description> Usage docs <marked> You can use markdown inside <marked> tag </marked> </item-description> <item-example-html> example html </item-example-html> ``` * map the page name to the html page created above (add an entry to `config.js`) ```js 'Page name' : 'styles/page-name.html', ``` The UI gallery is maintained by **Glen Hunter** with assistance from **Neil Donewar** and **Andrey Marushkevych**.
