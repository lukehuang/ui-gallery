Welcome to the Component Gallery

How to add a new component to the gallery

* create an html page using the following structure (see template.html) ```html <item-description> Usage docs <marked> You can use markdown inside <marked> tag </marked> </item-description> <item-example-html> example html </item-example-html> ``` * map the component name to the html page created above (add an entry to `config.js`) ```js 'component-name' : 'components/component-name.html', ``` * (only for js components) add necessary imports and angular module dependencies in `config.js`

Components vs Directives

In the gallery we distinguish between Components and Directives not by implementation but by usage: #### Components Components are independent widgets represented by an html tag. For example here's the 'iq-checkbox' component: ```html <iq-checkbox></iq-checkbox> ``` Internally components can be implemented as an angular directive. #### Directives Directives are attributes that are added to existing tags/components to enhance or add functionality. Here is an example using the 'load-wrapper' directive: ```html <div load-wrapper>Content</div> ```
